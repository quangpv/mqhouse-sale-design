# Thông tin cá nhân (User Info)

## 1. Screen Overview
- **Screen name**: Thông tin cá nhân / User Info
- **Purpose**: View and edit personal profile details — name, phone, avatar, operating area (districts/wards).
- **Business context**: Users manage their own profile information here. The operating area selection determines which districts the user (typically a Sale agent) serves. Account code is read-only and displayed for reference.

## 2. UI Structure
### Layout sections
- **Header**: Left arrow (back to Profile), title "Thông tin cá nhân".
- **Profile section**: Circular avatar with camera change icon, role badge "Quản trị viên" (primary blue), display name "Nguyễn Minh Thu", account code "ACC-2026-89A" in `textSecondary`.
- **Phone input**: Tel-type input, pre-filled "0912345678", with phone icon.
- **Operating area section**: Map pin icon header, province display "Quận 1, Thành phố Hồ Chí Minh", sub-district (ward) chips: Bến Nghé, Bến Thành, Nguyễn Cư Trinh, Đa Kao, Tân Định.
- **Sticky bottom bar**: "Cập nhật thông tin" button with check-circle save icon.
- **Success toast**: Checkmark icon + "Cập nhật thông tin thành công!", auto-hides after 3s (`setTimeout` in `triggerUpdate()`).

### Components
- **Back button** (arrow-left icon, `onclick` → navigate to Profile).
- **Circular avatar** with camera overlay (same pattern as Profile screen).
- **Role badge**: primary-blue pill.
- **Account code label**: grey-secondary text, non-editable.
- **Phone input**: `type="tel"`, pre-filled, with phone icon.
- **District chips**: round pill buttons (`rounded-full`), multi-select toggle, primary-blue when selected.
- **Sticky bottom bar**: fixed to bottom, contains primary save button.
- **Success toast**: fixed positioned, appears on `triggerUpdate()`, disappears after 3s.

### Important UI patterns
- Sticky bottom action bar overlays content — content area must have bottom padding.
- Toast notification is a simple show/hide with `setTimeout`; no animation defined.
- Chips use toggle select pattern (click to select/deselect).

## 3. Entry Points
- Profile → "Thông tin cá nhân".

## 4. States
- **Loading**: Not implemented in HTML mockup.
- **Empty**: Not implemented. Phone input is pre-filled; operating area chips are pre-populated.
- **Error**: Not implemented in HTML mockup.
- **Success**: Toast "Cập nhật thông tin thành công!" triggered by `triggerUpdate()`.

## 5. Business Rules
### Validation rules
- Phone: must be valid Vietnamese phone number (10-11 digits, starting with 0).
- Display name: required, max length not specified.
- At least one operating area (ward) must be selected.

### Permission rules
- Users can only edit their own profile.
- Account code is read-only and auto-generated by the system.
- Avatar upload may have file size/format constraints (10MB max, jpg/png/webp per Spec.md).

### Conditional rendering rules
- Ward chips toggle select state on click.
- Save button always visible and enabled.
- Role badge and account code always visible.
